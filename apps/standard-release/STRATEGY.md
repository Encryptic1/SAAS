# Standard Release — Product Strategy

## Product
GitHub release notes generator. Connect a repo, click generate, get markdown release notes from merged pull requests since your last tag. Edit, then publish.

## Viral Loop
- **Within-org**: release notes are shared internally and externally (changelog pages, blog posts)
- **Cross-org**: public changelogs reference "Generated by Standard Release"
- **Distribution**: GitHub Marketplace listing, dev tooling SEO ("release notes generator", "changelog automation")

## Pricing
| Tier | Price | Limits | Badge |
|------|-------|--------|-------|
| Free | $0 | 1 repo · 5 generations/mo | Yes (required) |
| Starter | $15/mo | Unlimited repos & generations | Removable |

## Unit Economics (targets)
- ARPU: $15
- Monthly churn: <6%
- LTV: ~$250 at $15 ARPU / 6% churn
- CAC: $0 (organic — GitHub Marketplace + SEO)

## KPIs
- Repos connected (activation)
- Generations per month (engagement)
- Notes published (conversion to "real" usage)
- Free → paid conversion rate

## FloodG8 / SyncDevTime synergy
- **FloodG8 Plan Editor**: import release notes into plan / reservoir metadata; attach to `parentPlanId` changelog
- **SyncDevTime**: time spent writing release notes is tracked
- **Standard Status**: failed deploy → "View release notes" deep link for rollback context

## Cross-sells
- **Inbound**: Standard Status failed deploy → "View release notes" deep link
- **Outbound**: published notes → "Insert snippet" picker for Standard Snippets (boilerplate changelog entries)

## 90-Day Milestones
1. **Week 1–2**: MVP live, 10 beta repos
2. **Week 3–4**: GitHub Marketplace listing submitted
3. **Month 2**: 100+ repos, first paid conversions
4. **Month 3**: GitHub App upgrade (PAT → App for multi-org install)
